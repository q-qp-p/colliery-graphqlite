## Triage notes

- **Crashes first.** [[GQLITE-T-0218]] is the only P0 — fix before any other
  TCK work, because crashes are uniquely bad (process death, undefined state,
  potential security implications) compared with wrong answers.
- **Fix the result decoder before deep-diving on `fail` scenarios.**
  [[GQLITE-T-0227]] (harness decoder) almost certainly inflates [[GQLITE-T-0223]]
  (expected empty got rows) and the bulk of the 479 "unmatched expected row"
  fails. Re-run the baseline after T-0227 to get a sharper picture before
  investing investigation time in T-0223 / T-0229.
- **Big-cluster fixes have outsized impact.** [[GQLITE-T-0219]] alone, if
  fixed cleanly, removes 69 error scenarios and may unblock T-0220 (which
  feels tangled with WITH-scoping).
- **Omnibus tracking for syntax.** [[GQLITE-T-0224]] is intentionally an
  umbrella — cutting 303 individual feature-requests would be noise. Split as
  sub-tickets when a cluster is picked up for implementation.
