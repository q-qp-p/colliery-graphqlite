# Add JSON arm to property access COALESCE chains in transform layer

## Parent Initiative

[[GQLITE-I-0030]] — Native JSON/Map/List Properties + Nested Access

## Objective

Add `node_props_json` / `edge_props_json` as a new arm in the property access COALESCE chains generated by `transform_property_access()`. This makes JSON-stored properties visible in RETURN, WHERE, and WITH clauses. Also update the `properties()` function and map projection transforms to include JSON properties.

| `src/backend/transform/transform_return.c` | Update `properties()` function transform and map projection `.*` to include JSON table in LEFT JOINs |

### Technical Approach

**COALESCE chain extension:** The existing chain is `COALESCE(text, int, real, bool)`. Add a fifth arm for JSON: `(SELECT npj.value FROM node_props_json npj JOIN property_keys pk ON npj.key_id = pk.id WHERE npj.node_id = n0.id AND pk.key = 'prop')`. Position it last since JSON is the newest type and least likely to match for existing data.

**Nested access already works:** `transform_property_access()` already handles `AST_NODE_PROPERTY` base recursively with `json_extract()`. Once the root property lookup includes the JSON table, `n.metadata.name` will naturally resolve to `json_extract(<coalesce_with_json>, '$.name')`.
